# 装配包络与设计域约束设计

## 1. 目标

把当前“组件必须留在现有组件包络内”的限制升级成“组件必须留在显式设计域内”。

这样做的原因是：

- 现有规则会天然阻止组件尺寸扩展
- 模型即使学会了步长限制，也可能持续撞到隐式包络墙
- 工程上更合理的做法是显式定义允许设计变化的装配域

## 2. 设计原则

- 设计域是状态的一部分，而不是运行时推导副产物
- 当前版本只支持一个轴对齐矩形设计域
- validator 只校验组件是否落在设计域内，不改变 mesh / physics 主逻辑
- baseline 默认给出一个略大于当前组件外包络的设计域

## 3. 状态结构

在 `ThermalDesignState.geometry` 下新增：

```yaml
geometry:
  type: multirect_2d
  design_domain:
    x0: 0.0
    y0: 0.0
    width: 1.2
    height: 0.5
```

含义：

- 所有组件矩形都必须位于这个矩形域内
- 组件可以在该域内部扩展、移动、收缩

## 4. 对 validator 的影响

把当前“当前组件包络”替换为“显式 design_domain”：

- 位置步长限制仍然按设计域宽高的 `25%`
- 组件是否越界按 `design_domain` 判断
- 不再因为高于当前 spreader 顶部就自动判非法，只要仍在 `design_domain` 内即可

## 5. baseline 建议

当前 baseline 组件最高到 `y=0.42`。

建议把首版设计域设为：

- `x0=0.0`
- `y0=0.0`
- `width=1.2`
- `height=0.5`

这样：

- 水平方向与当前底板对齐
- 竖直方向给 spreader 和后续小幅扩展留出余量
- 不会无限放开几何增长

## 6. 预期效果

加上设计域后，像 `heat_spreader.height: 0.1 -> 0.14` 这类提案，如果仍在设计域内，就不应再因为“超出当前组件包络”而被拒绝。

这会让 LLM 从“撞隐式墙”转成“在显式装配空间里优化”。
