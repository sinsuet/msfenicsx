## 1. 本补充文档的定位

已有的进度报告一到报告四讲清了：

- 报告一：问题结构（15 组件、32 维、2 目标、4 条硬约束）

但是这些报告里并没有把 `s1_typical` benchmark 本身的物理参数完整列出来，也没有展开两个优化目标的**数学定义**。老师在听汇报时通常会想追问：

- 到底每个组件发多少热？

本补充文档把这些内容补齐。所有数值都与仓库中当前 `scenarios/templates/s1_typical.yaml` 和 `scenarios/evaluation/s1_typical_eval.yaml` 对应，可以直接交叉核对。

## 2. 面板与坐标系

| 项目 | 数值/设置 |
| --- | --- |
| 坐标系 | `panel_xy`（二维面板平面） |
| 面板宽度 | `W = 1.0 m` |
| 面板高度 | `H = 0.8 m` |
| 面板面积 | `|Ω| = 0.8 m²` |
| 主布置区域 `main-deck` | `x ∈ [0.07, 0.93]`, `y ∈ [0.06, 0.69]` |
| 顶部禁布带 `top-edge-harness-strip` | `y ∈ [0.74, 0.80]` |

顶部 `y ∈ [0.69, 0.74]` 是预留给顶部散热窗口及其周边的走线缓冲，组件不能侵入 `y ≥ 0.74` 的顶部带，这保证散热窗口位置优化时不会被组件遮挡。

## 3. 材料参数

面板体由两种材料构成：

| material_id | 热导率 `k` [W/(m·K)] | 发射率 `ε` |
| --- | ---: | ---: |
| `panel_substrate`（面板基材） | `3.0` | `0.78` |
| `electronics_housing`（15 个组件） | `12.0` | `0.82` |

物理上的含义是：

- 组件区域热导率是基材的 4 倍，因此组件本身更像是“导热好的发热贴片”，主要热阻来自基材
- 组件发射率略高于基材，因此它们向散热边界的辐射耦合略强

## 4. 15 个组件：功率与发热密度

当前 `s1_typical` 是单工况 benchmark，所有 15 个组件的总功率和布置关系都固定。组件总功率合计 `114.0 W`，分布到 `1.0 m × 0.8 m` 的面板上，平均表面热流约 `142.5 W/m²`。

每个组件的发热并不是均匀铺满整个组件矩形，而是只在其**子发热斑块**上生效，大小由 `source_area_ratio` 决定：

```text
q_local = total_power / (source_area_ratio * component_area)
```

这意味着小比例组件（如 `c02` 的 `0.12`）代表**局部功率密度非常高的发热点**；大比例组件（如 `c14` 的 `0.48`）代表**相对均匀的温和热源**。

这解释了为什么 `c02 / c04 / c06 / c12` 是布局优化中的关键热源：它们既是高功率模块，又具有极高的局部热流密度，哪怕位置稍稍离开顶部散热窗口都会显著推高峰值温度。

### 4.3 源项在求解器中的形式

在 FEniCSx 求解器里，热源被表示为体积源项 `q(x)`：

```text
q(x) = total_power / source_area,  x ∈ source_polygon
q(x) = 0,                           otherwise
```

其中 `source_polygon` 是组件内按 `source_area_ratio` 缩比得到的子区域。对应实现见 `core/solver/physics_builder.py:_source_interpolator`。

## 5. 散热边界

面板上的散热由两部分组成：

1. 顶部**线段散热窗口**（可设计）
2. 其它外边界上的**背景对流 + 低发射率辐射冷却**（固定）

### 5.1 顶部散热窗口 `sink-top-window`

| 参数 | 数值 |
| --- | ---: |
| 类型 | `line_sink` |
| 所在边 | `top` (`y = 0.8`) |
| 散热面目标温度 `T_sink` | `286.0 K` |
| 对流换热系数 `h_sink` | `10.0 W/(m²·K)` |
| 发射率 `ε_sink` | `0.78`（未显式给出时，求解器默认继承 `panel_substrate.emissivity`） |
| 允许跨度范围 `span` | `[0.31, 0.69]` |
| 当前硬约束 | `total_radiator_span ≤ 0.48` |

散热窗口通过两个决策变量 `sink_start`、`sink_end` 定义区间 `[sink_start, sink_end]`，并将该段顶边作为内置的辐射 + 对流耦合边界。本 benchmark 只有这一段散热窗口，面板其他三条边以及顶边上未被窗口覆盖的部分进入下方的“背景边界”。

### 5.2 背景边界冷却 `background_boundary_cooling`

| 参数 | 数值 |
| --- | ---: |
| 环境温度 `T_amb` | `292.0 K` |
| 对流换热系数 `h_bg` | `0.15 W/(m²·K)` |
| 发射率 `ε_bg` | `0.08` |

这里的物理设定是：卫星面板除了顶部设计窗口之外，周围边界与环境的对流耦合很弱（`0.15`），辐射也很低（`0.08`）。因此设计变量里**顶部散热窗口的位置和跨度是主要的散热通道**，背景边界只是二次补偿。

### 5.3 Stefan–Boltzmann 常数

辐射项使用标准值：

```text
σ = 5.670374419e-8 W / (m² · K⁴)
```

见 `core/solver/case_to_geometry.py` 与 `core/solver/physics_builder.py`。

## 6. 控制方程

FEniCSx 求解稳态导热方程，带非线性辐射-对流散热边界：

```text
-∇·( k(x) ∇T(x) ) = q(x),   x ∈ Ω
```

边界条件按段拼合：

- 顶部散热窗口 `Γ_sink`：

  ```text
  -k ∇T · n = h_sink (T - T_sink) + ε_sink σ (T⁴ - T_sink⁴)
  ```

- 背景边界 `Γ_bg`（除 `Γ_sink` 外的所有外边界）：

  ```text
  -k ∇T · n = h_bg (T - T_amb) + ε_bg σ (T⁴ - T_amb⁴)
  ```

其中：

- `k(x)` 在组件区域为 `12.0`，基材区域为 `3.0`
- `q(x)` 在每个组件的 `source_polygon` 内取 `total_power / source_area`，其它位置为零
- `T_sink = 286 K`, `T_amb = 292 K`

这是一个典型的**稳态非线性导热/辐射耦合问题**，用 SNES 非线性求解器求解（`solver_profile.nonlinear_solver: snes`，容差 `1e-8`，最多 50 次迭代）。数值离散使用 `Lagrange` 一阶元，网格为 `nx=48, ny=40`。

## 7. 两个优化目标的精确定义

本 benchmark 的两个目标（`summary.temperature_max` 与 `summary.temperature_gradient_rms`）都是在求解得到的离散温度场 `T_h` 上计算的。它们的数学含义如下。

### 7.1 目标一：峰值温度 `summary.temperature_max`

```text
T_max = max_{x ∈ Ω} T_h(x)
```

在实现上等价于取有限元解在所有自由度上的节点最大值。这个指标直接衡量“最热点有多热”。

### 7.2 目标二：梯度 RMS `summary.temperature_gradient_rms`

**定义（精确）**：

```text
grad_rms = sqrt( (1 / |Ω|) · ∫_Ω |∇T_h(x)|² dx )
```

其中：

- `|Ω| = 0.8 m²` 是面板域面积
- `|∇T_h(x)|² = (∂T/∂x)² + (∂T/∂y)²` 是该点温度梯度的平方模
- 积分在整个 `Ω` 上进行，使用有限元变分形式完成

在实现上对应 `core/solver/gradient_metrics.py:compute_temperature_gradient_rms`，其 UFL 表达是：

```text
energy_density = inner(grad(T_h), grad(T_h)) * dx
integral       = assemble_scalar(form(energy_density))
grad_rms       = sqrt( max(integral, 0) / |Ω| )
```

### 7.3 这一目标的物理含义

- 量纲上是 `K/m`
- 反映的是**整个面板上温度在空间中有多“陡”**
- 它不是峰值温度附近的局部梯度，也不是某条截线的梯度，而是**整板平均的 L² 范数**
- 与“温度差”`T_max − T_min` 的区别：`grad_rms` 衡量温度场的空间变化率，即使 `T_max − T_min` 相同，越集中在小范围内的温差给出的 `grad_rms` 越大

所以把它称作“梯度优化”时，它优化的是**整板温度场的 L² 梯度能量密度的均方根**，而不是简单的最高温度或温度跨度。

### 7.4 两个目标的物理取舍

两个目标经常存在竞争：

- 让热点远离彼此、发散开 → `T_max` 下降，但场内温差分布更广 → `grad_rms` 可能上升
- 让所有热源集中靠近顶部散热窗口 → 顶部附近温差大 → `grad_rms` 可能上升，但 `T_max` 下降
- 让组件更均匀布满面板 → 场变化平缓 → `grad_rms` 下降，但关键热点附近局部可能不如顶部布局那样被压低 → `T_max` 可能上升

这正是使用 NSGA-II 做多目标搜索的动机：这两个目标无法通过一次标量加权清晰地表达“谁更好”。

## 8. 硬约束与其物理意义

在前面章节数值的基础上，当前 `s1_typical_eval` 定义的 4 条硬约束为：

| constraint | metric | 含义 |
| --- | --- | --- |
| `radiator_span_budget` | `case.total_radiator_span ≤ 0.48` | 散热窗口总跨度不能超过顶边的 `48%`，即散热面积预算有限 |
| `c01_peak_temperature_limit` | `component.c01-001.temperature_max ≤ 356 K` | 逻辑板不能超温 |
| `c08_peak_temperature_limit` | `component.c08-001.temperature_max ≤ 352 K` | I/O 板不能超温（略严于 `c01`） |
| `panel_temperature_spread_limit` | `components.max_temperature_spread ≤ 95 K` | 所有组件平均温度之间的最大差不能超过 `95 K` |

前三条约束直接绑定到 PDE 解；第四条 `panel_temperature_spread_limit` 是组件间的横向公平性约束，防止为了压低某个组件而让其它组件温度失控。

## 10. 小结

把本补充文档的内容压成一段可以直接拿去汇报的话：

> `s1_typical` 是一个 `1.0 m × 0.8 m` 的二维面板，热导率 `3.0 / 12.0 W/(m·K)`，15 个固定组件共发热 `114 W`，功率最高的 `c02 / c04 / c06 / c12` 在组件内部子斑块上的局部热流可达 `4 000 – 8 000 W/(m²)`。环境温度 `292 K`，顶部可设计的单段散热窗口以 `T_sink = 286 K`、`h = 10 W/(m²·K)` 的强耦合向外散热，其它边界几乎不散热（`h = 0.15, ε = 0.08`）。求解器在 `48 × 40` 网格、一阶 Lagrange 元、SNES 非线性迭代下求稳态温度场，然后计算两个优化目标：一个是全域 `T_max`；另一个是整板 L² 梯度均方根 `grad_rms = sqrt((1/|Ω|) ∫ |∇T|² dx)`，量纲 `K/m`。所谓“梯度优化”优化的就是这一整板梯度能量均方根，而不是某点梯度或温差。报告四的 `T_max / grad_rms / radiator_span` 比较，就是在这组固定物理参数下对 `raw / union / llm` 三条路线的直接量化对比。
